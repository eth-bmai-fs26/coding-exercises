What is Agentic AI?

An AI agent is a system that can:

  • Perceive its environment
  • Reason about what to do next
  • Act using tools and APIs
  • Learn from the results

Unlike a chatbot that answers one question, an agent pursues goals over multiple steps, deciding its own actions along the way.

Question → Answer

The LLM is the brain — it reads observations and history, then outputs a structured tool call.

Anatomy of an Agent

🧠 LLM (The Brain)

Receives a system prompt (role, goals, rules) and conversation history (observations, past actions, results). Outputs a structured tool call.

🔧 Tools

Concrete actions the agent can take: move(), talk(), search(), create_file(), API calls, database queries, etc.

📋 Memory / History

Running log of observations, actions, and results. Lets the agent build context and avoid repeating mistakes.

🌍 Environment

The world the agent operates in: a game, a codebase, a database, a web app, a robot's sensors. Provides observations and responds to actions.

Why Agentic AI?

Real-world applications

  • Software engineering — autonomous coding, debugging, testing
  • Customer support — handle multi-step resolutions
  • Data analysis — explore, query, visualize autonomously
  • Research — literature review, experiment design
  • Operations — supply chain, scheduling, monitoring

Why now?

  • LLMs are finally good enough at reasoning to drive multi-step workflows
  • Tool use / function calling standardized across providers
  • Frameworks like LangChain, CrewAI, Claude Agent SDK make it accessible
  • The gap between "chat" and "do" is closing fast
"The next frontier isn't smarter models — it's models that can take action."

Agent Tools

The agent has 4 tools it can call. scan() runs automatically every turn.

move(direction="north|south|east|west")

Move one step. Triggers cell entry events.

talk(message="your message")

Converse with NPCs. Keywords trigger quest items.

collect()

Pick up items in current cell (dossiers, weapons, materials).

fabricate(item="item name")

Craft weapons or sell salvage at Forge/Lab.

scan() — automatic, every turn

Reports current position, adjacent cells, and what's visible. The agent receives this as an observation.

Tool Call Format

The LLM must output exactly one tool call in this format:

TOOL: move(direction="north")
TOOL: talk(message="Do you have any work for me?")
TOOL: collect()
TOOL: fabricate(item="Flamethrower")

The engine parses it with a simple regex:

def parse_tool_call(text: str) -> tuple[str, dict]:
    """Extract tool name and arguments from LLM output."""
    match = re.search(r'TOOL:\s*(\w+)\((.*?)\)', text, re.DOTALL)
    tool_name = match.group(1)

    args = {}
    for kv in re.finditer(r'(\w+)\s*=\s*["\']([^"\']*)["\']', match.group(2)):
        args[kv.group(1)] = kv.group(2)

    return tool_name, args

The LLM can reason freely in text before the TOOL: line — only the tool call is extracted.

    Steps:
    1. Build a system message (role, goals, tools, rules)
    2. Build a user message (status, history, observations)
    3. Call client.models.generate_content()
    4. Return response.text  →  must contain "TOOL: ..."
    """

Input

  • operative — player state (health, dossiers, inventory, position)
  • world — game world (grid, NPCs, quest flags)
  • history — list of {role, content} dicts
  • client — Gemini API client

Output

  • A string containing exactly one tool call:
TOOL: move(direction="north")

The engine extracts and executes it.

Building the Prompt

Step 1: System Message

Tell the LLM who it is, what it can do, and how to respond:

system_msg = f"""You are Agent Lambda, a spy on Isla Perdida.

OBJECTIVE: Collect 10 dossiers of classified intelligence.

AVAILABLE TOOLS:
{TOOLS_DESCRIPTION}

RULES:
- Respond with EXACTLY ONE tool call: TOOL: name(args)
- Talk to every informant — they reveal quests and items.
- Collect items before leaving a cell.
- Craft weapons BEFORE entering robot cells.
- Track what you've already explored and don't revisit empty cells.
"""

Key insight: The quality of this system prompt directly determines how well your agent plays. This is where prompt engineering matters most.

Building the Prompt

Step 2: User Message (Context Window)

# Current state
status = operative.status_text()
# "Health: 3/3 | Dossiers: 2/10 | Inventory: [USB Drive] | Pos: (5,3)"

# Recent history (last ~10-15 entries)
recent = history[-15:]
transcript = "\n".join(
    f"{h['role'].upper()}: {h['content'][:200]}"
    for h in recent
)

# Journal (key discoveries from NPC conversations)
journal = "\n".join(operative.journal[-5:])

user_msg = f"""Current Status: {status}

Journal (key intel):
{journal}

Recent Events:
{transcript}

What is your next action? Respond with TOOL: ..."""

Tip: Truncate history to avoid exceeding token limits. The journal preserves key discoveries even when old turns scroll out.

Building the Prompt

Step 3: Call the LLM

import google.generativeai as genai

response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=user_msg,
    config=genai.types.GenerateContentConfig(
        system_instruction=system_msg,
        max_output_tokens=500,
    ),
)

return response.text
# → "Let me check the forge for crafting options.\n\nTOOL: move(direction=\"west\")"

That's it! Three components: system prompt + user message + API call. The engine handles everything else — parsing, execution, display, logging.

The Improvement Cycle

The Debrief System

After each run, the notebook generates a paste block with your code + game log:

# In your notebook:
generate_debrief_paste()

This outputs a formatted text block containing:

  • Your current think_llm() code
  • Game outcome (win/loss/timeout)
  • Turn-by-turn action log
  • Key statistics (turns, dossiers, health)
  • Game-specific context (map, tools, quests)

How to use it

1 Run the debrief cell
2 Copy the paste block
3 Paste into ChatGPT (or Gemini)
4 ChatGPT acts as a coach
5 Apply its suggestions to your prompt
6 Re-run and iterate

ChatGPT as Your Coach

The debrief instructs ChatGPT to follow a coaching protocol:

1. DIAGNOSE

Analyze the game log. What went wrong? Where did the agent get stuck, loop, or make bad decisions?

2. GUIDE

Ask targeted questions: "What happens when the agent reaches a robot without a weapon? How could the prompt prevent this?"

3. HINT

Suggest strategies, not code: "Consider adding a rule about weapon requirements before entering robot cells."

4. IMPLEMENT TOGETHER

Help translate the idea into prompt changes. Never gives a complete drop-in solution — you build it together.

The coach is designed to teach you prompt engineering, not to solve the game for you.

Common Agent Failures & Fixes

SymptomRoot CausePrompt Fix
Agent walks in circles No exploration strategy Add "prioritize unvisited cells" + track visited in prompt
Dies to robot Enters robot cell without weapon Add rule: "NEVER enter 🤖 without the required weapon"
Never talks to NPCs Prompt doesn't emphasize talk() Add "Talk to EVERY informant — they give quests"
Runs out of turns Inefficient exploration Add priority ordering: "collect nearby → quest → explore"
Forgets quest items Key info scrolls out of history Include operative.journal in user message
Invalid tool calls LLM outputs wrong format Add explicit format example in system prompt

Prompt Engineering Tips

System Prompt

  • Be specific — "collect 10 dossiers" not "complete the mission"
  • List all tools with exact syntax and examples
  • Add rules for dangerous situations (robots, traps)
  • Define priorities — what to do first, second, third
  • Include format examples of valid tool calls

User Message

  • Always include current status (health, dossiers, position)
  • Truncate history to last ~15 entries
  • Include the journal — it preserves NPC discoveries
  • End with a clear ask: "What is your next action?"
  • Keep it structured — labeled sections help the LLM

The secret: A good agent isn't about clever code — it's about a clear, well-structured prompt that gives the LLM the right context to make good decisions.
