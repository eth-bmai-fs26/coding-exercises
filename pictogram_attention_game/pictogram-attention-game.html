Attention Reveal

You were running
attention all along.

"You just ran an attention mechanism. Not a metaphor of one · the actual computation. Every time you read a scrambled sentence and correctly assigned a companion to its noun, you solved the same problem a transformer solves thousands of times per token: given this piece of information, who in the context is it relevant to?"

What you were actually doing

Routing by compatibility, not by position. You never used the location of a symbol to decide what it modified. You used its identity · which family it belonged to. That is exactly what attention does: it computes a compatibility score between every pair of tokens and routes information based on that score, ignoring sequence order.
Separating "who" from "what". Each companion symbol carries two distinct things: an address (which noun should receive this?) and a payload (what property does it describe?). You kept these separate intuitively. In a transformer, the address is encoded in the Key vector and the payload is encoded in the Value vector.
The averaging trap. In the bonus question, the circle collected two conflicting color signals and the result was undefined. This is not a bug in the game · it is a real limitation of the mechanism. When multiple tokens attend to the same target with conflicting values, the output averages them. This is why multi-head attention exists.

Attention is a function that takes a set of tokens and produces, for each token, a weighted summary of the other tokens it finds relevant. It has three steps.

Step 1 · Score

Every token is projected into three spaces: a Query, a Key, and a Value. To compute how much token i should attend to token j, we take the dot product of i's Query with j's Key.

score(i, j) = Qᵢ · Kⱼ

In the game, ◈ (square) has Query Q=(1,0). The companion ≋ (big) has Key K=(1,0). Their dot product is 1 · a perfect match. ⌇ (tall) has Key K=(0,1). Its dot product with ◈ is 0 · no match. The score encodes compatibility.

Step 2 · Normalize

Raw scores are passed through a softmax, which converts them into a probability distribution summing to 1. Tokens with higher scores receive higher weights.

weights = softmax(scores / √d)

The √d scaling (where d is the dimension of the Key vectors) prevents the dot products from becoming too large and pushing the softmax into a near-zero gradient region. In the game, our scores are already 0 or 1, so this step is trivial · but in real models with d=64 or d=512, it matters.

Step 3 · Aggregate

The output for token i is a weighted sum of the Value vectors of all other tokens, using the normalized weights from Step 2.

output(i) = Σⱼ weight(i,j) × Vⱼ

In the game, the square ◈ attended to ≋ and ⧉ with equal weight 0.5. Its output was 0.5 × V(big) + 0.5 × V(red). The Value vector is the payload · the actual information being transferred.

Why this matters

Attention is permutation-equivariant: if you shuffle the input tokens, the outputs shuffle in the same way but their content does not change. This is precisely what you experienced in Phase 4 · scrambling the symbols did not change the meaning because attention does not care about order. Positional encodings are added separately to inject order when it is needed.

Each token is projected into three vectors by three learned weight matrices: W_Q, W_K, and W_V. In the game, these projections are hardcoded · but in a real transformer they are learned from data.

Reading the table

Nouns have non-zero Queries, zero Keys. They ask questions but do not answer them. The square asks "is this a square companion?" with Q=(1,0). The circle asks "is this a circle companion?" with Q=(0,1).
Companions have non-zero Keys, zero Queries. They carry an address but do not seek information themselves. A square companion has K=(1,0): it answers "yes" only when a square (Q=(1,0)) asks. A circle companion has K=(0,1): it answers "yes" only when a circle asks.
Values encode the payload. The Value vector holds the actual information. BIG encodes (+1, 0, 0, 0): it increases the square's size dimension by 1. RED encodes (0, -1, 0, 0): it sets the square's color dimension to -1. These are arbitrary choices for this game · in real models, Value vectors are dense and learned.

Reading: no square info · circ_shape = +0.5 (tallish) · circ_color = +0.5 (orangeish)

Why the values are ±0.5 and not ±1

Each noun attended to two companions with equal weight 0.5. The output is an average, not a sum. With only one companion, the output would be ±1. With three companions of equal weight, each contributes ±0.33. This averaging is a core property of attention · and it is why multi-head attention was invented. Different heads can specialize on different property dimensions, preventing the dilution.
