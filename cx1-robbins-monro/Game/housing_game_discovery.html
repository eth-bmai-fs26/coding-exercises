Phase 4a: Simple Fixed-Step Algorithm

First attempt at automation - adjust by a fixed amount

🧮 Simple Fixed-Step Rule

Let's start with a simple approach:

If Ŷ < Y: Wnew = Wold + α
If Ŷ > Y: Wnew = Wold - α

Logic: If prediction is too low, increase W by α.
If prediction is too high, decrease W by α.

🤔 What's the Problem?

This rule always adjusts by the same amount (α), regardless of how far off we are.

Think: What might happen when we get close to the target?

Answer: When we get close to the target, the fixed step size causes oscillation! The algorithm keeps overshooting because it always moves by α, even when the error is tiny. It bounces back and forth around the target instead of settling precisely.

This is why we need proportional updates (Robbins-Monro) that adjust by α × error - smaller errors lead to smaller adjustments!

Phase 4b: Robbins-Monro Algorithm

The smart solution - proportional adjustments!

📊 Clean Data Environment

In this phase, all house prices follow the exact formula: Price = 40 × Area + 50

There is no noise - every data point lies perfectly on the line. This lets us see if the Robbins-Monro algorithm can converge exactly to W = 40.

🎯 Expected outcome: With the right α, W should converge precisely to 40.0!

🤖 Robbins-Monro Solution

Instead of fixed steps, adjust by an amount proportional to the error:

Wnew = Wold - α × (Prediction - Actual)

Key Difference: When far from target → big adjustment.
When close to target → small adjustment.
At the target → zero adjustment!

With clean data and the right α, this converges exactly to W = 40!

🤔 Quick Check #2

Question: Why does α (learning rate) need to be small in this problem?

Hint: Think about the update formula Wnew = Wold - α × (Prediction - Actual)

Answer: The error naturally includes the house area when you expand it!

Let's expand the formula:

1️⃣ Prediction = W × Area + 50

2️⃣ Actual = 40 × Area + 50 (true W = 40)

3️⃣ Error = Prediction - Actual = (W - 40) × Area

Example: If W = 30 and we see a house with Area = 40:

  • Error = (30 - 40) × 40 = -400
  • With α = 0.1: Update = 0.1 × 400 = 40
  • W changes by 40, jumping from 30 to 70 (way past 40!)

Key insight: Bigger houses create bigger updates. To keep updates reasonable, α must be small enough that α × max_area < 1, so α < 1/40 ≈ 0.025.
