\begin{frame}{What is Agentic AI?}
\begin{columns}[T]
\begin{column}{0.52\textwidth}
An \hl{AI agent} is a system that can:
\begin{itemize}
  \item \textbf{Perceive} its environment
  \item \textbf{Reason} about what to do next
  \item \textbf{Act} using tools and APIs
  \item \textbf{Learn} from the results
\end{itemize}
\vspace{0.8em}
Unlike a chatbot that answers one question, an agent \hl{pursues goals over multiple steps}, deciding its own actions along the way.
\end{column}
\begin{column}{0.42\textwidth}
\centering

\begin{frame}{From Chatbot to Agent}
\centering
\small
\begin{tabular}{@{}l l l@{}}
\toprule
 & \textbf{Chatbot} & \textbf{Agent} \\
\midrule
\textbf{Interaction}    & Single turn Q\&A            & Multi-turn, goal-directed \\
\textbf{Who decides?}   & Human gives each instruction & LLM decides its own next step \\
\textbf{Tools}          & None (text only)             & Can call functions, APIs, tools \\
\textbf{Memory}         & Conversation context         & Structured history + state tracking \\
\textbf{Failure mode}   & Wrong answer                 & Wrong action (real consequences) \\
\textbf{Output}         & Text response                & Structured tool call $\rightarrow$ execution \\
\bottomrule
\end{tabular}

\vspace{1em}
The key shift: the LLM output is not the final answer -- \hl{it's an instruction that gets executed}.
\end{frame}

% TOOL CALL FORMAT
% ----------------------------------------------------------
\begin{frame}[fragile]{Tool Call Format}
The LLM must output \textbf{exactly one} tool call in this format:
\begin{lstlisting}[style=textcode]
TOOL: move(direction="north")
TOOL: talk(message="Do you have any work for me?")
TOOL: collect()
TOOL: fabricate(item="Flamethrower")
\end{lstlisting}
\vspace{0.3em}
The engine parses it with a simple regex:
\begin{lstlisting}[style=pycode]
def parse_tool_call(text: str) -> tuple[str, dict]:
    """Extract tool name and arguments from LLM output."""
    pattern = r'TOOL:\s*(\w+)\((.*?)\)'
    match = re.search(pattern, text, re.DOTALL)
    tool_name = match.group(1)

    args = {}
    for kv in re.finditer(r'(\w+)\s*=\s*"([^"]*)"', match.group(2)):
        args[kv.group(1)] = kv.group(2)

    return tool_name, args
\end{lstlisting}
\mutedtext{\scriptsize The LLM can reason freely in text before the \texttt{TOOL:} line -- only the tool call is extracted.}
\end{frame}

% BUILDING THE PROMPT: SYSTEM MESSAGE

\begin{frame}[fragile]{Building the Prompt -- Step 1: System Message}
Tell the LLM \textbf{who it is}, \textbf{what it can do}, and \textbf{how to respond}:
\begin{lstlisting}[style=pycode]
system_msg = f"""You are Agent Lambda, a spy on Isla Perdida.

OBJECTIVE: Collect 10 dossiers of classified intelligence.

AVAILABLE TOOLS:
{TOOLS_DESCRIPTION}

RULES:
- Respond with EXACTLY ONE tool call: TOOL: name(args)
- Talk to every informant - they reveal quests and items.
- Collect items before leaving a cell.
- Craft weapons BEFORE entering robot cells.
- Track what you've already explored and don't revisit empty cells.
"""
\end{lstlisting}
\vspace{0.3em}
\mutedtext{\scriptsize \hl{Key insight:} The quality of this system prompt \textbf{directly determines} how well your agent plays. This is where prompt engineering matters most.}
\end{frame}

% ----------------------------------------------------------
% BUILDING THE PROMPT: USER MESSAGE
% ----------------------------------------------------------
\begin{frame}[fragile]{Building the Prompt -- Step 2: User Message}
\begin{lstlisting}[style=pycode]
# Current state
status = operative.status_text()
# "Health: 3/3 | Dossiers: 2/10 | Inventory: [USB Drive] | Pos: (5,3)"

# Recent history (last ~10-15 entries)
recent = history[-15:]
transcript = "\n".join(
    f"{h['role'].upper()}: {h['content'][:200]}"
    for h in recent
)

# Journal (key discoveries from NPC conversations)
journal = "\n".join(operative.journal[-5:])

user_msg = f"""Current Status: {status}

Journal (key intel):
{journal}

Recent Events:
{transcript}

What is your next action? Respond with TOOL: ..."""
\end{lstlisting}
\mutedtext{\scriptsize \hl{Tip:} Truncate history to avoid exceeding token limits. The journal preserves key discoveries even when old turns scroll out.}
\end{frame}

% ----------------------------------------------------------
% BUILDING THE PROMPT: API CALL
% ----------------------------------------------------------
\begin{frame}[fragile]{Building the Prompt -- Step 3: Call the LLM}
\begin{lstlisting}[style=pycode]
import google.generativeai as genai

response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=user_msg,
    config=genai.types.GenerateContentConfig(
        system_instruction=system_msg,
        max_output_tokens=500,
    ),
)

return response.text
# -> "Let me check the forge...\n\nTOOL: move(direction=\"west\")"
\end{lstlisting}
\vspace{0.5em}
\begin{center}
\fcolorbox{ethteal}{ethteal!8}{\parbox{12.5cm}{\centering\small\color{textmain}%
\textbf{That's it!} Three components: system prompt + user message + API call.
The engine handles everything else -- parsing, execution, display, logging.}}
\end{center}
\end{frame}

% ----------------------------------------------------------
% PROMPT ENGINEERING TIPS
% ----------------------------------------------------------
\begin{frame}{Prompt Engineering Tips}
\begin{columns}[T]
\begin{column}{0.48\textwidth}
{\color{ethteal}\textbf{System Prompt}}
\begin{itemize}\setlength\itemsep{2pt}
  \item \textbf{Be specific} -- ``collect 10 dossiers'' not ``complete the mission''
  \item \textbf{List all tools} with exact syntax and examples
  \item \textbf{Add rules} for dangerous situations (robots, traps)
  \item \textbf{Define priorities} -- what to do first, second, third
  \item \textbf{Include format examples} of valid tool calls
\end{itemize}
\end{column}
\begin{column}{0.48\textwidth}
{\color{ethteal}\textbf{User Message}}
\begin{itemize}\setlength\itemsep{2pt}
  \item \textbf{Always include current status} (health, dossiers, position)
  \item \textbf{Truncate history} to last \textasciitilde15 entries
  \item \textbf{Include the journal} -- it preserves NPC discoveries
  \item \textbf{End with a clear ask}: ``What is your next action?''
  \item \textbf{Keep it structured} -- labeled sections help the LLM
\end{itemize}
\end{column}
\end{columns}
\vspace{0.5em}
\begin{center}
\fcolorbox{ethteal}{ethteal!8}{\parbox{12.5cm}{\centering\small\color{textmain}%
\textbf{The secret:} A good agent isn't about clever code -- it's about a \hl{clear, well-structured prompt} that gives the LLM the right context to make good decisions.}}
\end{center}
\end{frame}
