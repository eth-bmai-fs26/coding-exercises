================================================================================
MINI-GAMES SPECIFICATION: Progressive Agentic AI Exercises
================================================================================

CONTEXT
-------
This spec is for building 4 mini-games that teach the skills needed to solve
the capstone "Quest Hero" exercise. Each mini-game isolates 1-2 agentic AI
skills. Students progress through them in order, and by the end they've
practiced every technique needed for the capstone.

All games share the same contract:
  - The student implements a `think(...)` function
  - The think function returns a string: TOOL: tool_name(arg="value")
  - The game engine calls think() each turn in a perceive -> think -> act loop
  - The LLM API is Google Gemini: client.models.generate_content(...)
  - Games run in Google Colab notebooks
  - The SDK is `google-genai` (import: `from google import genai`)

The repository is at:
  https://github.com/eth-bmai-fs26/coding-exercises (branch: week3)
  Directory: coding-exercises/agentic_ai_hero/

The capstone game (Quest Hero) already exists and is fully functional.
The mini-games should be added alongside it, each in its own subdirectory.


================================================================================
CAPSTONE REFERENCE: QUEST HERO
================================================================================

Quest Hero is an 8x8 RPG grid world. The hero must collect 10 gold by:
- Picking up 3 treasures (1 gold each)
- Completing 2 delivery errands (2 gold each) by talking to NPCs/innkeepers
- Crafting 2 weapons from materials found in forests + gold at shops
- Fighting 2 dragons with the correct weapons (3 gold each)
- Selling dragon loot at shops (1 gold each)

The hero has: hearts (HP), gold, inventory, position, visited set, journal.
9 tools: look, move, talk, pick_up, buy, use, fight, rest, status.
Limited visibility (only adjacent cells visible).
NPC dialogue is powered by an LLM oracle (provided, not student-implemented).

The student implements two think functions:
  1. think_rule_based(hero, world, history) -> str   (if/else logic)
  2. think_llm(hero, world, history, client) -> str   (LLM-powered)

Key skills needed for a successful think_llm:
  a) Structured output: LLM must emit TOOL: name(arg="val"), not prose
  b) State injection: prompt includes position, inventory, surroundings
  c) Exploration: track visited cells, prefer unvisited
  d) History management: select relevant recent turns, avoid noise
  e) Self-correction: detect stuck loops, inject warnings
  f) NPC interpretation: parse cryptic clues into actionable plans
  g) Multi-step planning: chain sub-goals (find ore -> smith -> craft -> fight)

ARCHITECTURE & CONVENTIONS
================================================================================

Each mini-game should follow the same pattern as Quest Hero:

1. A Python package (e.g., `mini1_locked_room/`) containing:
   - Game engine module(s) (world, hero/player state, tools)
   - An agent module with:
     - TOOLS_DESCRIPTION string
     - parse_tool_call(text) function (reuse from Quest Hero or simplify)
     - run_agent() game loop
     - think function stub (raises NotImplementedError)
   - Display module (simple print-based is fine; HTML optional)

2. A student notebook: `mini1_locked_room_student.ipynb`
   - Setup cell (pip install, imports)
   - Exploration section (try tools manually)
   - TODO: implement think function
   - Run cell
   - Game log download cell

3. A solutions notebook: `mini1_locked_room_solutions.ipynb`

Tool call format (same across all games):
  TOOL: tool_name(arg="value")

Gemini API call pattern (same across all games):
  from google import genai
  response = client.models.generate_content(
      model="gemini-2.5-flash",
      contents=user_message,
      config=genai.types.GenerateContentConfig(
          system_instruction=system_message,
          max_output_tokens=300,
      ),
  )
  return response.text

Game log format (JSON, same structure):
  {outcome, final_score, total_turns, turns: [{turn, state, observation, action, result, success}, ...]}

Each game should save a log file that students can download and inspect.

IMPORTANT: Keep each mini-game self-contained. Do NOT import from quest_hero.
Each mini-game should have its own small engine. The games should be simple
enough that the engine fits in ~100-200 lines total.

IMPORTANT: For mini-games 1 and 2, the student should implement the think
function ONLY using rule-based if/else logic (no LLM). These games teach
the perceive-think-act loop and state management. Mini-games 3 and 4
introduce the LLM.


================================================================================
MINI-GAME 1: "The Locked Room"
================================================================================

SKILL FOCUS: Structured output format + the perceive-think-act loop

SETTING:
  A single room with 3 interactable objects. No grid, no movement.
  The hero must escape the room by performing the right sequence of actions.

GAME MECHANICS:
  - Room contains: a locked chest, a painting on the wall, a dusty shelf
  - Behind the painting: a key
  - Key opens the chest -> chest contains a golden lever
  - Pulling the lever opens a hidden door -> escape!
  - 3-step solution: examine("painting") -> pick_up("key") -> use("key")
    (or similar — keep it to exactly 3 required actions)
  - Max turns: 10

TOOLS (simplified set):
  TOOL: examine(object="painting|chest|shelf|door")
    Look closely at an object. Reveals hidden items or details.
  TOOL: pick_up(object="key|lever|...")
    Pick up an item you've discovered.
  TOOL: use(object="key|lever|...")
    Use an item from your inventory on something in the room.

WHAT STUDENTS IMPLEMENT:
  think(room_state, inventory, history) -> str

  - room_state: dict describing visible objects and their states
    e.g., {"painting": "A dusty painting hangs on the wall.",
           "chest": "A locked wooden chest sits in the corner.",
           "shelf": "A dusty shelf with old books.",
           "door": "A heavy iron door. It won't budge."}
  - inventory: list of item names the player holds
  - history: list of {role, content} dicts from previous turns

  Students write if/else logic. No LLM needed.

WHAT STUDENTS LEARN:
  - The perceive-think-act loop pattern
  - Returning exactly one TOOL: call per turn
  - Reading game state and history to decide the next action
  - The parse_tool_call() mechanism

EXPECTED SOLUTION (rule-based, ~15 lines):
  def think(room_state, inventory, history):
      if "key" not in inventory:
          # Check if we've examined the painting yet
          examined_painting = any("painting" in h["content"] for h in history if h["role"] == "result")
          if not examined_painting:
              return 'TOOL: examine(object="painting")'
          return 'TOOL: pick_up(object="key")'
      if "key" in inventory:
          return 'TOOL: use(object="key")'
      return 'TOOL: examine(object="chest")'

FAILURE MODE TO EXPERIENCE:
  Without checking history, the student's agent will examine the painting
  every turn forever. This teaches: "your agent has no memory unless you
  give it memory via the history parameter."


================================================================================
MINI-GAME 2: "The Treasure Trail"
================================================================================

SKILL FOCUS: Exploration + visited tracking + multi-turn state management

SETTING:
  A 1x8 corridor (linear). 3 treasures hidden at fixed positions.
  Hero has limited visibility (can only see 1 cell ahead in each direction).
  Collect all 3 treasures to win.

GAME MECHANICS:
  - 8 cells in a row, indexed 0-7
  - Hero starts at position 0
  - Treasures at positions 2, 5, 7 (fixed, not random)
  - Hero can see current cell + 1 cell left + 1 cell right
  - Max turns: 20
  - Win condition: pick up all 3 treasures

TOOLS:
  TOOL: move(direction="left|right")
    Move one step left or right. Fails if out of bounds.
  TOOL: pick_up()
    Pick up treasure at current position. Fails if none.
  TOOL: look()
    See current cell and neighbors (automatically called each turn).

WHAT STUDENTS IMPLEMENT:
  think(position, visited, treasures_collected, observation, history) -> str

  - position: int (0-7)
  - visited: set of ints (positions visited)
  - treasures_collected: int (0-3)
  - observation: str (what you see: current cell + neighbors)
  - history: list of {role, content} dicts

  Students write if/else logic. No LLM needed.

WHAT STUDENTS LEARN:
  - Using visited set to avoid revisiting cells
  - Systematic exploration (always move toward unvisited cells)
  - Multi-turn decision making based on accumulated state
  - The concept of "the agent has no memory — you must track state explicitly"

EXPECTED SOLUTION (rule-based, ~15 lines):
  def think(position, visited, treasures_collected, observation, history):
      # Pick up treasure if available
      if "treasure" in observation.lower() and "current" in observation.lower():
          last_result = [h for h in history if h["role"] == "result"]
          if not last_result or "nothing" not in last_result[-1]["content"]:
              return 'TOOL: pick_up()'
      # Move toward unvisited cells, prefer right (toward end)
      for target in range(8):
          if target not in visited:
              if target > position:
                  return 'TOOL: move(direction="right")'
              else:
                  return 'TOOL: move(direction="left")'
      return 'TOOL: move(direction="right")'

FAILURE MODE TO EXPERIENCE:
  Without visited tracking, the agent oscillates left-right endlessly.
  This teaches: "you must track where you've been to explore efficiently."

MINI-GAME 3: "The Riddler's Maze"
================================================================================

SKILL FOCUS: LLM interpretation of natural language clues + goal-directed action

SETTING:
  A 4x4 grid. 2 NPCs give cryptic clues about a hidden treasure's location.
  Hero must talk to NPCs, interpret their clues, navigate to the treasure,
  and pick it up. First game where the student uses an LLM.

GAME MECHANICS:
  - 4x4 grid (16 cells)
  - Hero starts at (3, 0) — bottom-left
  - NPC 1 at (1, 0): gives a clue about the row
    e.g., "The gold rests where the eagles nest — high above the ground."
    (meaning: row 0, the top)
  - NPC 2 at (3, 3): gives a clue about the column
    e.g., "Seek where the sun sets, at the edge of the western sky."
    (Wait — that would be confusing. Let's use:)
    e.g., "The treasure hides two steps from the eastern wall."
    (meaning: column 1... or just make the clues clearer in-world)
  - Treasure at (0, 2) — revealed only by combining both clues
  - Limited visibility: hero sees adjacent cells only
  - Max turns: 25
  - Win condition: pick up the treasure
  - 1 mountain (impassable) to force routing decisions

  NPC responses use a simple stub oracle (canned responses, not LLM-powered).
  The STUDENT's think function uses the LLM to interpret the clues.

TOOLS:
  TOOL: move(direction="north|south|east|west")
  TOOL: talk(question="your question")
  TOOL: pick_up()
  TOOL: look()  (auto-called each turn)

WHAT STUDENTS IMPLEMENT:
  think_llm(position, visited, inventory, observation, history, journal, client) -> str

  This is the FIRST game where students call the LLM:
  - Build a system message explaining the game and available tools
  - Build a user message with position, visited set, observation, history, journal
  - Call client.models.generate_content(...)
  - Return response.text

WHAT STUDENTS LEARN:
  - Constructing a system prompt that constrains LLM output to TOOL: format
  - Including game state (position, visited, journal) in the user message
  - The LLM can interpret "high above the ground" as "row 0" — something
    if/else logic cannot do
  - Including NPC dialogue (journal) in the prompt so the LLM can reason about it

EXPECTED SOLUTION (~30 lines):
  def think_llm(position, visited, inventory, observation, history, journal, client):
      system = f"""You are an agent navigating a 4x4 grid to find hidden treasure.
  Available tools: move(direction="north|south|east|west"), talk(question="..."), pick_up()
  NEVER call look(). You see your surroundings automatically.
  NPCs give cryptic clues about the treasure location. Interpret them!
  The grid is 4x4: rows 0(top)-3(bottom), columns 0(left)-3(right).
  Respond with exactly one TOOL: call."""

      journal_text = "\n".join(journal) if journal else "(no NPC conversations yet)"
      visited_str = ", ".join(str(p) for p in sorted(visited))

      user_msg = f"""Position: {position}
  Visited: {visited_str}
  Observation: {observation}
  Journal (NPC clues): {journal_text}
  Recent history: {history[-10:]}
  Find the treasure! Interpret NPC clues to figure out where it is."""

      response = client.models.generate_content(
          model="gemini-2.5-flash",
          contents=user_msg,
          config=genai.types.GenerateContentConfig(
              system_instruction=system, max_output_tokens=300,
          ),
      )
      return response.text

FAILURE MODE TO EXPERIENCE:
  Without journal entries in the prompt, the LLM wanders randomly because
  it never "hears" the NPC clues. This teaches: "what you include in the
  prompt determines the agent's intelligence."

MINI-GAME 4: "The Dragon's Errand"
================================================================================

SKILL FOCUS: Multi-step planning + resource management + self-correction

SETTING:
  A 4x4 grid with a forest (has ore), a shop (crafts a sword), and a dragon.
  To win: find ore -> go to shop -> craft sword (costs 2 gold) -> fight dragon.
  Hero starts with 3 gold. One wrong fight = death.

GAME MECHANICS:
  - 4x4 grid
  - Hero starts at (3, 0) with 3 gold, 2 hearts
  - Forest at (1, 0): contains "Ember Ore" (pick_up)
  - Shop at (3, 3): crafts "Fire Sword" from Ember Ore + 2 gold
  - Dragon at (0, 3): fight with Fire Sword = win (victory!), without = death
  - 1 NPC at (2, 2): gives a hint about the quest chain
    "The beast fears only forged flame. The smith in the southeast
     can make what you need — but bring him burning stone from the
     western woods."
  - 1 mountain at (1, 2) to force routing
  - Max turns: 30
  - Win condition: defeat the dragon
  - Loss condition: hearts reach 0 (fighting without sword, or running out of turns)

TOOLS:
  TOOL: move(direction="north|south|east|west")
  TOOL: talk(question="your question")
  TOOL: pick_up()
  TOOL: buy(item="Fire Sword")  — at shop, needs Ember Ore + 2 gold
  TOOL: fight()  — at dragon cell, needs Fire Sword or lose 1 heart
  TOOL: look()  (auto-called each turn)

WHAT STUDENTS IMPLEMENT:
  think_llm(hero_state, observation, history, journal, client) -> str

  hero_state is a dict: {position, hearts, gold, inventory, visited}

  Students must write a prompt that:
  - Explains the multi-step quest chain
  - Includes current state (gold, inventory, hearts)
  - Includes journal (NPC hint)
  - Detects being stuck (same position for N turns)
  - Warns against fighting without a weapon

WHAT STUDENTS LEARN:
  - Prompting for multi-step planning: the LLM must maintain a goal chain
  - Resource math in prompts: "you have 3 gold, sword costs 2"
  - Stuck detection: inject warnings when the agent loops
  - Negative instructions: "NEVER fight without a weapon or you die"
  - The LLM needs strategic guidance, not just state — it must understand
    prerequisites and ordering

EXPECTED SOLUTION (~40 lines):
  def think_llm(hero_state, observation, history, journal, client):
      system = f"""You are an agent in a 4x4 RPG grid. Your goal: defeat the dragon.

  QUEST CHAIN (you MUST follow this order):
  1. Go to the forest and pick up Ember Ore
  2. Go to the shop and buy Fire Sword (costs Ember Ore + 2 gold)
  3. Go to the dragon and fight it WITH the Fire Sword

  CRITICAL: NEVER fight the dragon without Fire Sword — you will DIE.
  NEVER call look() or status() — you see your state every turn.

  Available tools: move(direction="..."), talk(question="..."), pick_up(),
                   buy(item="..."), fight()

  Grid: 4x4, rows 0(top)-3(bottom), cols 0(left)-3(right).
  Respond with exactly one TOOL: call."""

      # Stuck detection
      stuck = ""
      pos = hero_state["position"]
      same = sum(1 for h in history[-6:] if h["role"]=="observation" and str(pos) in h["content"])
      if same >= 3:
          stuck = f"\nWARNING: You've been at {pos} for {same} turns. MOVE NOW.\n"

      user_msg = f"""Position: {pos}  Hearts: {hero_state['hearts']}  Gold: {hero_state['gold']}
  Inventory: {hero_state['inventory']}
  Visited: {sorted(hero_state['visited'])}
  {stuck}
  Observation: {observation}
  Journal: {journal}
  History (last 8): {history[-8:]}
  What's your next move?"""

      response = client.models.generate_content(
          model="gemini-2.5-flash",
          contents=user_msg,
          config=genai.types.GenerateContentConfig(
              system_instruction=system, max_output_tokens=300,
          ),
      )
      return response.text

FAILURE MODE TO EXPERIENCE:
  Without explicit "NEVER fight without a weapon" in the prompt, the LLM
  charges at the dragon immediately and dies. This teaches: "the LLM needs
  strategic constraints, not just information."

  Without stuck detection, the agent may loop at the shop trying to buy
  without ore. This teaches: "self-correction mechanisms are essential."

================================================================================
IMPLEMENTATION NOTES
================================================================================

1. SELF-CONTAINED: Each mini-game must be fully self-contained. Do NOT import
   from quest_hero or from other mini-games. Students should be able to run
   any mini-game independently.

2. SIMPLE ENGINES: Keep game engines minimal. No complex inheritance or
   abstractions. A single engine.py file per game is ideal.

3. CONSISTENT API: All think functions return "TOOL: name(arg=\"val\")".
   All games use parse_tool_call() (can be copied into each agent.py).

4. GAME LOGS: Every game should save a JSON log file (same pattern as Quest
   Hero) so students can debug turn-by-turn.

5. COLAB-READY: Notebooks should work in Google Colab with the same setup
   pattern (git clone/pull + pip install).

6. GRADUAL COMPLEXITY: The think function signature grows across games:
   - Game 1: think(room_state, inventory, history)
   - Game 2: think(position, visited, treasures_collected, observation, history)
   - Game 3: think_llm(position, visited, inventory, observation, history, journal, client)
   - Game 4: think_llm(hero_state, observation, history, journal, client)
   - Capstone: think_llm(hero, world, history, client)

7. STUDENT NOTEBOOK STRUCTURE:
   Each student notebook should have:
   - A "Learning Goal" markdown cell at the top explaining what skill this
     game teaches and why it matters for agentic AI
   - An exploration section where students try tools manually
   - Clear TODO markers
   - Hints section (collapsible if possible)
   - A reflection question at the end

8. NO RANDOM ELEMENTS: All games should be deterministic (fixed treasure
   locations, fixed NPC positions). This ensures reproducible debugging.

9. DISPLAY: Use simple print-based display (grid rendering with emojis).
   HTML display is nice-to-have but not required for mini-games.

--- quest_hero/hero.py ---

from dataclasses import dataclass, field

@dataclass
class Hero:
    hearts: int = 3
    gold: int = 0
    inventory: list[str] = field(default_factory=list)
    position: tuple[int, int] = (7, 0)
    visited: set[tuple[int, int]] = field(default_factory=lambda: {(7, 0)})
    journal: list[str] = field(default_factory=list)
    MAX_HEARTS = 3
    WIN_GOLD = 10

    def has_item(self, name: str) -> bool:
        return name in self.inventory

    def add_item(self, name: str) -> None:
        self.inventory.append(name)

    def remove_item(self, name: str) -> bool:
        if name in self.inventory:
            self.inventory.remove(name)
            return True
        return False

    def add_gold(self, amount: int) -> None:
        self.gold += amount

    def spend_gold(self, amount: int) -> bool:
        if self.gold >= amount:
            self.gold -= amount
            return True
        return False

    def take_damage(self, amount: int) -> None:
        self.hearts = max(0, self.hearts - amount)

    def heal(self, amount: int) -> None:
        self.hearts = min(self.MAX_HEARTS, self.hearts + amount)

    @property
    def is_alive(self) -> bool:
        return self.hearts > 0

    @property
    def has_won(self) -> bool:
        return self.gold >= self.WIN_GOLD and self.is_alive

    def status_text(self) -> str:
        inv = ", ".join(self.inventory) if self.inventory else "empty"
        return (
            f"Hearts: {self.hearts}/{self.MAX_HEARTS} | "
            f"Gold: {self.gold} | "
            f"Position: {self.position} | "
            f"Inventory: [{inv}] | "
            f"Visited: {len(self.visited)} cells"
        )


--- quest_hero/agent.py (TOOLS_DESCRIPTION + parse_tool_call + run_agent) ---

TOOLS_DESCRIPTION = """Available tools (use exactly one per turn):

TOOL: look()
  See the types of adjacent cells (north/south/east/west). Free action.

TOOL: move(direction="north|south|east|west")
  Move one step in a cardinal direction.

TOOL: talk(question="your question here")
  Talk to an NPC, innkeeper, or shopkeeper in your current cell.

TOOL: pick_up()
  Pick up items in your current cell.

TOOL: buy(item="item name")
  Buy an item or craft a weapon at the current shop.

TOOL: use(item="item name")
  Use a consumable item: Bread (+1 heart), Health Potion (+2 hearts).

TOOL: fight()
  Fight the monster in your current cell.

TOOL: rest()
  Rest at an inn. Costs 1 gold, restores 1 heart.

TOOL: status()
  Check your current hearts, gold, inventory, and position."""


def parse_tool_call(text: str) -> tuple[str, dict]:
    import re
    match = re.search(r'TOOL:\s*(\w+)\((.*?)\)', text, re.DOTALL)
    if not match:
        simple = re.search(r'TOOL:\s*(\w+)', text)
        if simple:
            return simple.group(1), {}
        return "look", {}
    tool_name = match.group(1)
    args_str = match.group(2).strip()
    if not args_str:
        return tool_name, {}
    args = {}
    for kv_match in re.finditer(r'(\w+)\s*=\s*["\']([^"\']*)["\']', args_str):
        args[kv_match.group(1)] = kv_match.group(2)
    return tool_name, args


--- quest_hero/tools.py (tool dispatch pattern) ---

@dataclass
class ToolResult:
    success: bool
    message: str

class GameTools:
    def execute(self, tool_name: str, args: dict) -> ToolResult:
        tool_map = {
            "look": lambda: self.look(),
            "move": lambda: self.move(args.get("direction", "")),
            "talk": lambda: self.talk(args.get("question", "")),
            "pick_up": lambda: self.pick_up(),
            "buy": lambda: self.buy(args.get("item", "")),
            "use": lambda: self.use(args.get("item", "")),
            "fight": lambda: self.fight(),
            "rest": lambda: self.rest(),
            "status": lambda: self.status(),
        }
        fn = tool_map.get(tool_name.lower().strip())
        if fn is None:
            return ToolResult(False, f"Unknown tool '{tool_name}'.")
        return fn()


--- quest_hero_solutions.ipynb: think_llm solution ---

def think_llm(hero, world, history, client):
    system = f"""You are an AI agent controlling a hero in an RPG grid world.
YOUR GOAL: Collect 10 gold and survive.
AVAILABLE TOOLS:
{TOOLS_DESCRIPTION}
CRITICAL RULES:
- Respond with exactly ONE tool call: TOOL: tool_name(arg="value")
- NEVER call look() or status(). They run automatically.
- If pick_up() fails, the cell is EMPTY — leave immediately.
EXPLORATION IS KEY:
- Explore the full 8x8 map. Prioritize UNVISITED cells.
GOLD STRATEGY (need 10):
- Treasures: 3 gold max. Not enough!
- Talk to every NPC — quests give 2 gold each.
- Craft weapons, fight dragons (3 gold each).
- Talk to innkeepers — they have errands.
Think step by step, then output EXACTLY one TOOL: line."""

    recent = history[-15:]
    history_text = "\n".join(f"  [{h['role']}] {h['content'][:200]}" for h in recent)
    journal_text = "\n".join(f"  - {e[:150]}" for e in hero.journal[-10:]) if hero.journal else "  (none)"
    row, col = hero.position
    cell = world.get_cell(row, col)
    visited_str = ", ".join(f"({r},{c})" for r, c in sorted(hero.visited))

    stuck_warning = ""
    same_pos_count = 0
    for h in reversed(history):
        if h["role"] == "observation" and f"position ({row}, {col})" in h["content"]:
            same_pos_count += 1
        elif h["role"] == "observation":
            break
    if same_pos_count >= 3:
        stuck_warning = f"\nWARNING: STUCK at ({row},{col}) for {same_pos_count} turns! MOVE NOW.\n"

    user_msg = f"""STATUS: {hero.status_text()}
CELL: {cell.description or cell.cell_type.label}
{stuck_warning}
VISITED ({len(hero.visited)}/64): {visited_str}
HISTORY:
{history_text}
JOURNAL:
{journal_text}
Output one TOOL: call."""

    response = client.models.generate_content(
        model="gemini-2.5-flash",
        contents=user_msg,
        config=genai.types.GenerateContentConfig(
            system_instruction=system, max_output_tokens=500,
        ),
    )
    return response.text
